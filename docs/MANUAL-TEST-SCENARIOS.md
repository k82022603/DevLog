## 테스트 준비

### 사전 조건

**1. 서비스 실행 확인**
```bash
# 모든 컨테이너가 Running 상태인지 확인
docker ps

# 다음 3개 컨테이너가 실행 중이어야 함:
# - devlog-frontend (포트 3000)
# - devlog-backend (포트 8080)
# - devlog-postgres (포트 5432)
```

**2. 브라우저 준비**
- Chrome 또는 Firefox 사용 권장
- 개발자 도구 열기 (F12)
- Console 탭과 Network 탭 확인 준비

**3. 초기 데이터 확인**
- 프로젝트 5개 존재
- 개발 로그 10개 존재
- 기술 태그 여러 개 존재

**4. 테스트 데이터 준비**
```
테스트용 프로젝트명: "테스트 프로젝트 A"
테스트용 로그 제목: "수동 테스트 로그"
테스트 날짜: 오늘 날짜
```

## 테스트 시나리오

### TS-001: 대시보드 조회 및 통계 확인

**목적**: 대시보드가 올바르게 로드되고 통계가 정확하게 표시되는지 확인

**테스트 단계**:
1. 브라우저에서 http://localhost:3000 접속
2. 페이지 로딩 확인
3. 통계 카드 확인
4. 최근 활동 로그 확인

**예상 결과**:
- 페이지가 3초 이내에 로드됨
- 통계 카드에 숫자가 표시됨
- 최근 로그 목록이 표시됨
- 에러 메시지가 없음

---

### TS-002: 프로젝트 전체 CRUD

**목적**: 프로젝트 생성, 조회, 수정, 삭제 기능이 정상 작동하는지 확인

**테스트 단계**: [하위 테스트 케이스 참조](#tc-002-1-프로젝트-생성)

**예상 결과**:
- 모든 CRUD 작업이 성공
- 데이터가 올바르게 저장되고 표시됨
- 삭제된 데이터가 목록에서 사라짐

---

### TS-003: 개발 로그 전체 CRUD

**목적**: 개발 로그 생성, 조회, 수정, 삭제 기능이 정상 작동하는지 확인

**테스트 단계**: [하위 테스트 케이스 참조](#tc-003-1-개발-로그-생성)

**예상 결과**:
- 모든 CRUD 작업이 성공
- 날짜/시간이 올바르게 저장됨
- 기분 상태가 정확히 표시됨

---

### TS-004: 검색 및 필터링

**목적**: 검색 및 필터링 기능이 정상 작동하는지 확인

**테스트 단계**: [하위 테스트 케이스 참조](#tc-004-1-프로젝트-검색)

**예상 결과**:

- 필터 조건이 올바르게 적용됨

---

### TS-005: 반응형 디자인

**목적**: 다양한 화면 크기에서 레이아웃이 올바르게 표시되는지 확인

**테스트 단계**: [하위 테스트 케이스 참조](#tc-005-1-데스크톱-뷰)

**예상 결과**:

- 터치/클릭이 정상 작동

**목적**: 잘못된 입력이나 네트워크 오류 시 적절한 에러 메시지가 표시되는지 확인

## TC-001: 대시보드 기능

### TC-001-1: 대시보드 페이지 로드

**테스트 ID**: TC-001-1
**우선순위**: High
**관련 시나리오**: TS-001

**전제 조건**:
- 브라우저가 열려있음
- 서비스가 실행 중

**테스트 단계**:
1. 주소창에 `http://localhost:3000` 입력
2. Enter 키 누름
3. 페이지 로딩 시간 측정
4. Console 탭에서 에러 확인

**예상 결과**:
- [ ] 페이지가 3초 이내에 로드됨
- [ ] "DevLog" 로고가 표시됨
- [ ] 네비게이션 메뉴가 표시됨 (대시보드, 로그, 프로젝트, 설정)
- [ ] Console에 에러가 없음

**실제 결과**:
```
기록란:
- 로딩 시간: ___초
- 에러 발생 여부: 예/아니오
- 에러 내용:
```

### TC-001-2: 통계 카드 표시

**테스트 ID**: TC-001-2
**우선순위**: High
**관련 시나리오**: TS-001

**전제 조건**:
- TC-001-1 통과
- 대시보드 페이지가 로드됨

**테스트 단계**:
1. 대시보드 상단의 통계 카드 영역 확인
2. 각 카드의 제목과 숫자 확인
3. 카드가 4개인지 확인 (전체 프로젝트, 활성 프로젝트, 전체 로그, 이번 주 로그)

**예상 결과**:
- [ ] 통계 카드가 4개 표시됨
- [ ] 각 카드에 아이콘이 표시됨
- [ ] 각 카드에 숫자가 표시됨 (0 이상)
- [ ] "전체 프로젝트" 카드에 5가 표시됨
- [ ] "전체 로그" 카드에 10이 표시됨

**실제 결과**:
```
기록란:
- 전체 프로젝트 수: ___
- 활성 프로젝트 수: ___
- 전체 로그 수: ___
- 이번 주 로그 수: ___
```

## TC-002: 프로젝트 기능

### TC-002-1: 프로젝트 생성

**테스트 ID**: TC-002-1
**우선순위**: High
**관련 시나리오**: TS-002

**전제 조건**:
- 로그인 완료 (현재는 인증 없음)
- 프로젝트 목록 페이지 접근 가능

**테스트 단계**:
1. 네비게이션에서 "프로젝트" 클릭
2. "새 프로젝트" 버튼 클릭
3. 프로젝트 정보 입력:
   - 이름: "테스트 프로젝트 A"
   - 설명: "수동 테스트를 위한 프로젝트입니다"
   - 상태: "ACTIVE" 선택
   - 시작일: 오늘 날짜
   - 진행률: 50
4. "저장" 버튼 클릭
5. 성공 메시지 확인
6. 프로젝트 목록으로 돌아가서 생성된 프로젝트 확인

**예상 결과**:
- [ ] 프로젝트 생성 폼이 표시됨
- [ ] 모든 필드가 입력 가능함
- [ ] "저장" 버튼 클릭 시 성공 토스트 메시지 표시
- [ ] 프로젝트 목록에 새 프로젝트가 추가됨
- [ ] 입력한 정보가 정확히 표시됨

**실제 결과**:
```
기록란:
- 생성 성공 여부: 예/아니오
- 토스트 메시지 내용:
- 목록에서 확인됨: 예/아니오
```

## TC-003: 개발 로그 기능

### TC-003-1: 개발 로그 생성

**테스트 ID**: TC-003-1
**우선순위**: High
**관련 시나리오**: TS-003

**전제 조건**:
- 프로젝트가 1개 이상 존재
- 로그 목록 페이지 접근 가능

**테스트 단계**:
1. 네비게이션에서 "로그" 클릭
2. "새 로그" 버튼 클릭
3. 로그 정보 입력:
   - 프로젝트: "DevLog 플랫폼" 선택
   - 날짜: 오늘 날짜
   - 시작 시간: 09:00
   - 종료 시간: 12:00
   - 제목: "수동 테스트 로그"
   - 설명: "수동 테스트를 위한 개발 로그입니다"
   - 성과: "테스트 케이스 작성 완료"
   - 도전 과제: "없음"
   - 학습 내용: "수동 테스트 방법 학습"
   - 기분: "GOOD" 선택
4. "저장" 버튼 클릭
5. 성공 메시지 확인
6. 로그 목록에서 생성된 로그 확인

**예상 결과**:
- [ ] 로그 생성 폼이 표시됨
- [ ] 프로젝트 선택 드롭다운이 작동함
- [ ] 날짜/시간 입력이 정상 작동함
- [ ] 기분 선택이 정상 작동함
- [ ] "저장" 버튼 클릭 시 성공 토스트 표시
- [ ] 로그 목록에 새 로그가 추가됨
- [ ] 작업 시간이 자동 계산됨 (3시간 = 180분)

**실제 결과**:
```
기록란:
- 생성 성공 여부: 예/아니오
- 작업 시간 계산: ___분
- 토스트 메시지:
```

## 테스트 팁

### 효율적인 테스트를 위한 팁

1. **우선순위 높은 테스트 먼저**
   - High 우선순위 테스트 케이스부터 시작
   - 핵심 기능이 작동하지 않으면 나머지 테스트 의미 없음

2. **브라우저 개발자 도구 활용**
   - Console 탭: JavaScript 에러 확인
   - Network 탭: API 요청/응답 확인
   - Elements 탭: DOM 구조 확인

3. **스크린샷 및 비디오 녹화**
   - 버그 발견 시 즉시 스크린샷
   - 재현이 어려운 버그는 화면 녹화

4. **다양한 브라우저에서 테스트**
   - Chrome, Firefox, Edge에서 각각 테스트
   - Safari (macOS/iOS)에서도 가능하면 테스트

5. **실제 사용 시나리오 고려**
   - 테스트 케이스에 없더라도 실제 사용자처럼 행동
   - 예상치 못한 사용 패턴 시도

6. **성능 모니터링**
   - 페이지 로딩 시간 측정
   - 메모리 사용량 확인
   - CPU 사용률 모니터링
